<span data-ttu-id="533d6-111">API 閘道位於用戶端和服務之間。</span><span class="sxs-lookup"><span data-stu-id="533d6-111">An API gateway sits between clients and services.</span></span> <span data-ttu-id="533d6-112">它會作為反向 Proxy，將要求從用戶端路由傳送到服務。</span><span class="sxs-lookup"><span data-stu-id="533d6-112">It acts as a reverse proxy, routing requests from clients to services.</span></span> <span data-ttu-id="533d6-113">它也會執行各種跨領域工作，例如驗證、SSL 終止和速率限制。</span><span class="sxs-lookup"><span data-stu-id="533d6-113">It may also perform various cross-cutting tasks such as authentication, SSL termination, and rate limiting.</span></span> <span data-ttu-id="533d6-114">如果您未部署閘道，用戶端就必須將要求直接傳送給前端服務。</span><span class="sxs-lookup"><span data-stu-id="533d6-114">If you don't deploy a gateway, clients must send requests directly to front-end services.</span></span> <span data-ttu-id="533d6-115">不過，直接向用戶端公開服務會有一些潛在問題：</span><span class="sxs-lookup"><span data-stu-id="533d6-115">However, there are some potential problems with exposing services directly to clients:</span></span>
