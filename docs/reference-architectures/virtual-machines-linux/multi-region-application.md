## <a name="recommendations"></a><span data-ttu-id="7136c-131">建議</span><span class="sxs-lookup"><span data-stu-id="7136c-131">Recommendations</span></span>

<span data-ttu-id="7136c-132">多區域架構可以提供比部署到單一區域更高的可用性。</span><span class="sxs-lookup"><span data-stu-id="7136c-132">A multi-region architecture can provide higher availability than deploying to a single region.</span></span> <span data-ttu-id="7136c-133">如果區域中斷會影響主要區域，您可以使用[流量管理員][traffic-manager]容錯移轉到次要區域。</span><span class="sxs-lookup"><span data-stu-id="7136c-133">If a regional outage affects the primary region, you can use [Traffic Manager][traffic-manager] to fail over to the secondary region.</span></span> <span data-ttu-id="7136c-134">如果應用程式的個別子系統失敗，此架構也可以提供協助。</span><span class="sxs-lookup"><span data-stu-id="7136c-134">This architecture can also help if an individual subsystem of the application fails.</span></span>

<span data-ttu-id="7136c-135">有數種一般方法可用來跨區域實現高可用性：</span><span class="sxs-lookup"><span data-stu-id="7136c-135">There are several general approaches to achieving high availability across regions:</span></span>   

* <span data-ttu-id="7136c-136">搭配熱待命的主動/被動。</span><span class="sxs-lookup"><span data-stu-id="7136c-136">Active/passive with hot standby.</span></span> <span data-ttu-id="7136c-137">流量會傳送到其中一個區域，而另一個區域會以熱待命模式等候。</span><span class="sxs-lookup"><span data-stu-id="7136c-137">Traffic goes to one region, while the other waits on hot standby.</span></span> <span data-ttu-id="7136c-138">熱待命表示次要區域中隨時都已配置 VM 且正在執行中。</span><span class="sxs-lookup"><span data-stu-id="7136c-138">Hot standby means the VMs in the secondary region are allocated and running at all times.</span></span>
* <span data-ttu-id="7136c-139">搭配冷待命的主動/被動。</span><span class="sxs-lookup"><span data-stu-id="7136c-139">Active/passive with cold standby.</span></span> <span data-ttu-id="7136c-140">流量會傳送到其中一個區域，而另一個區域會以冷待命模式等候。</span><span class="sxs-lookup"><span data-stu-id="7136c-140">Traffic goes to one region, while the other waits on cold standby.</span></span> <span data-ttu-id="7136c-141">冷待命表示在需要容錯移轉之前，不會在次要區域中配置 VM。</span><span class="sxs-lookup"><span data-stu-id="7136c-141">Cold standby means the VMs in the secondary region are not allocated until needed for failover.</span></span> <span data-ttu-id="7136c-142">此方式的執行成本較小，但在失敗發生時通常需要較久的時間才能上線。</span><span class="sxs-lookup"><span data-stu-id="7136c-142">This approach costs less to run, but will generally take longer to come online during a failure.</span></span>

<span data-ttu-id="7136c-146">此架構著重於搭配熱待命的主動/被動 (使用流量管理員進行容錯移轉)。</span><span class="sxs-lookup"><span data-stu-id="7136c-146">This architecture focuses on active/passive with hot standby, using Traffic Manager for failover.</span></span> <span data-ttu-id="7136c-147">請注意，您可以部署少量的 VM 來進行熱待命，然後視需要相應放大。</span><span class="sxs-lookup"><span data-stu-id="7136c-147">Note that you could deploy a small number of VMs for hot standby and then scale out as needed.</span></span>


### <a name="regional-pairing"></a><span data-ttu-id="7136c-148">區域配對</span><span class="sxs-lookup"><span data-stu-id="7136c-148">Regional pairing</span></span>
