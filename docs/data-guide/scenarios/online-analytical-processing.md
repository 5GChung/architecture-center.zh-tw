# <a name="online-analytical-processing-olap"></a><span data-ttu-id="8f8e5-102">線上分析處理 (OLAP)</span><span class="sxs-lookup"><span data-stu-id="8f8e5-102">Online analytical processing (OLAP)</span></span>

<span data-ttu-id="8f8e5-103">線上分析處理 (OLAP) 是一種可組織大型企業資料庫和支援複雜分析的技術。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-103">Online analytical processing (OLAP) is a technology that organizes large business databases and supports complex analysis.</span></span> <span data-ttu-id="8f8e5-104">它可用來執行複雜的分析查詢，且不會對交易系統造成負面影響。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-104">It can be used to perform complex analytical queries without negatively affecting transactional systems.</span></span>

<span data-ttu-id="8f8e5-105">企業用來儲存其所有交易和記錄的資料庫，稱為[線上交易處理 (OLTP)](online-transaction-processing.md) 資料庫。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-105">The databases that a business uses to store all its transactions and records are called [online transaction processing (OLTP)](online-transaction-processing.md) databases.</span></span> <span data-ttu-id="8f8e5-106">這些資料庫通常具有逐一輸入的記錄。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-106">These databases usually have records that are entered one at a time.</span></span> <span data-ttu-id="8f8e5-107">它們常包含大量對組織有價值的資訊。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-107">Often they contain a great deal of information that is valuable to the organization.</span></span> <span data-ttu-id="8f8e5-108">不過，用於 OLTP 的資料庫並不是針對分析而設計的。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-108">The databases that are used for OLTP, however, were not designed for analysis.</span></span> <span data-ttu-id="8f8e5-109">因此，要從這些資料庫擷取答案，必須耗費許多時間與精力。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-109">Therefore, retrieving answers from these databases is costly in terms of time and effort.</span></span> <span data-ttu-id="8f8e5-110">OLAP 系統的目的是要協助您以高效能的方式從資料中擷取這項商業智慧資訊。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-110">OLAP systems were designed to help extract this business intelligence information from the data in a highly performant way.</span></span> <span data-ttu-id="8f8e5-111">這是因為，OLAP 資料庫針對大量讀取、低量寫入工作負載而進行了最佳化。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-111">This is because OLAP databases are optimized for heavy read, low write workloads.</span></span>

## <a name="when-to-use-this-solution"></a><span data-ttu-id="8f8e5-113">使用此解決方案的時機</span><span class="sxs-lookup"><span data-stu-id="8f8e5-113">When to use this solution</span></span>

<span data-ttu-id="8f8e5-114">在下列情況下請考慮使用 OLAP：</span><span class="sxs-lookup"><span data-stu-id="8f8e5-114">Consider OLAP in the following scenarios:</span></span>

- <span data-ttu-id="8f8e5-115">您需要快速執行複雜的分析與特定查詢，且不會對 OLTP 系統造成負面影響。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-115">You need to execute complex analytical and ad hoc queries rapidly, without negatively affecting your OLTP systems.</span></span> 
- <span data-ttu-id="8f8e5-116">您想要為商業使用者提供簡單的方式，從您的資料產生報表</span><span class="sxs-lookup"><span data-stu-id="8f8e5-116">You want to provide business users with a simple way to generate reports from your data</span></span>
- <span data-ttu-id="8f8e5-117">您想要提供多種彙總方式，讓使用者能夠取得快速且一致的結果。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-117">You want to provide a number of aggregations that will allow users to get fast, consistent results.</span></span> 

<span data-ttu-id="8f8e5-118">為大量資料套用彙總計算時，OLAP 特別具有效益。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-118">OLAP is especially useful for applying aggregate calculations over large amounts of data.</span></span> <span data-ttu-id="8f8e5-119">OLAP 系統針對大量讀取案例進行了最佳化，例如分析和商業智慧。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-119">OLAP systems are optimized for read-heavy scenarios, such as analytics and business intelligence.</span></span> <span data-ttu-id="8f8e5-120">OLAP 可讓使用者將多維度資料切分成能夠二維檢視的配量 (例如樞紐分析表)，或是依特定值篩選資料。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-120">OLAP allows users to segment multi-dimensional data into slices that can be viewed in two dimensions (such as a pivot table) or filter the data by specific values.</span></span> <span data-ttu-id="8f8e5-121">此程序有時也稱為「切割與細分」資料，無論資料是否跨數個資料來源進行分割，都可以執行。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-121">This process is sometimes called "slicing and dicing" the data, and can be done regardless of whether the data is partitioned across several data sources.</span></span> <span data-ttu-id="8f8e5-122">這有助於使用者找出趨勢和模式以及瀏覽資料，而無須得知傳統資料分析的詳細資料。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-122">This helps users to find trends, spot patterns, and explore the data without having to know the details of traditional data analysis.</span></span>

<span data-ttu-id="8f8e5-123">[語意模型](../concepts/semantic-modeling.md)可協助商業使用者抽離關聯的複雜性，而更輕鬆快速地分析資料。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-123">[Semantic models](../concepts/semantic-modeling.md) can help business users abstract relationship complexities and make it easier to analyze data quickly.</span></span>

## <a name="challenges"></a><span data-ttu-id="8f8e5-124">挑戰</span><span class="sxs-lookup"><span data-stu-id="8f8e5-124">Challenges</span></span>

<span data-ttu-id="8f8e5-125">在獲得 OLAP 系統提供的所有優點之前，有若干難題有待克服：</span><span class="sxs-lookup"><span data-stu-id="8f8e5-125">For all the benefits OLAP systems provide, they do produce a few challenges:</span></span>

- <span data-ttu-id="8f8e5-126">雖然 OLTP 系統中的資料會透過從各種來源流入的交易持續更新，但 OLAP 資料存放區通常會根據商業需求，以頻率遠低於此的間隔重新整理。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-126">Whereas data in OLTP systems is constantly updated through transactions flowing in from various sources, OLAP data stores are typically refreshed at a much slower intervals, depending on business needs.</span></span> <span data-ttu-id="8f8e5-127">這表示 OLAP 系統較適用於策略性商業決策，而不是對變更的立即回應。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-127">This means OLAP systems are better suited for strategic business decisions, rather than immediate responses to changes.</span></span> <span data-ttu-id="8f8e5-128">此外，也必須規劃某種程度的資料清理和協調流程，讓 OLAP 資料存放區保持在最新狀態。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-128">Also, some level of data cleansing and orchestration needs to be planned to keep the OLAP data stores up-to-date.</span></span>
- <span data-ttu-id="8f8e5-129">不同於出現在 OLTP 系統中的傳統正規化關聯式資料表，OLAP 資料模型通常是多維度的。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-129">Unlike traditional, normalized, relational tables found in OLTP systems, OLAP data models tend to be multidimensional.</span></span> <span data-ttu-id="8f8e5-130">這樣就不容易甚至無法直接對應至實體關聯性或物件導向模型，因為每個屬性會對應至一個資料行。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-130">This makes it difficult or impossible to directly map to entity-relationship or object-oriented models, where each attribute is mapped to one column.</span></span> <span data-ttu-id="8f8e5-131">相對地，OLAP 系統通常會以星形或雪花式結構描述來取代傳統的正規化。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-131">Instead, OLAP systems typically use a star or snowflake schema in place of traditional normalization.</span></span>

<span data-ttu-id="8f8e5-133">在 Azure 中，保存在 OLTP 系統 (例如 Azure SQL Database) 中的資料會複製到 OLAP 系統中，例如 [Azure Analysis Services](/azure/analysis-services/analysis-services-overview)。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-133">In Azure, data held in OLTP systems such as Azure SQL Database is copied into the OLAP system, such as [Azure Analysis Services](/azure/analysis-services/analysis-services-overview).</span></span> <span data-ttu-id="8f8e5-134">資料探索和視覺效果工具 (像是 [Power BI](https://powerbi.microsoft.com)、Excel 和第三方工具) 會連線至 Analysis Services 伺服器，並且可讓使用者以高度互動和豐富的視覺化方式深入了解模型資料。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-134">Data exploration and visualization tools like [Power BI](https://powerbi.microsoft.com), Excel, and third-party options connect to Analysis Services servers and provide users with highly interactive and visually rich insights into the modeled data.</span></span> <span data-ttu-id="8f8e5-135">從 OLTP 到 OLAP 的資料流程通常會使用 SQL Server Integration Services 進行協調，而這些服務可使用 [Azure Data Factory](/azure/data-factory/concepts-integration-runtime) 來執行。</span><span class="sxs-lookup"><span data-stu-id="8f8e5-135">The flow of data from OLTP data to OLAP is typically orchestrated using SQL Server Integration Services, which can be executed using [Azure Data Factory](/azure/data-factory/concepts-integration-runtime).</span></span>
