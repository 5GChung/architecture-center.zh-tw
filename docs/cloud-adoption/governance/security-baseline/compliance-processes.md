## <a name="planning-review-and-reporting-processes"></a><span data-ttu-id="1ebc5-108">規劃、檢閱及報告流程</span><span class="sxs-lookup"><span data-stu-id="1ebc5-108">Planning, review, and reporting processes</span></span>

<span data-ttu-id="1ebc5-109">即使是雲端中最佳的安全性基準工具，其效果也就和其所支援的流程與原則相同而已。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-109">The best Security Baseline tools in the cloud are only as good as the processes and policies that they support.</span></span> <span data-ttu-id="1ebc5-110">以下是一組安全性基準專業領域經常會牽涉到的範例流程。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-110">The following is a set of example processes commonly involved in the Security Baseline discipline.</span></span> <span data-ttu-id="1ebc5-111">當您規劃可讓您根據業務變更和 IT 小組 (賦有將治理指引付諸行動的工作) 意見反應來繼續更新安全性原則的流程時，請使用這些範例作為起點。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-111">Use these examples as a starting point when planning the processes that will allow you to continue to update security policy based on business change and feedback from the security and IT teams tasked with turning governance guidance into action.</span></span>

<span data-ttu-id="1ebc5-112">**初始風險評估及規劃**：在您初始採用安全性基準專業領域時，識別您的核心業務風險和與雲端安全性相關的承受度。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-112">**Initial risk assessment and planning**: As part of your initial adoption of the Security Baseline discipline, identify your core business risks and tolerances related to cloud security.</span></span> <span data-ttu-id="1ebc5-113">使用此資訊與您的 IT 和安全性小組成員討論特定技術風險，以及開發一組基本的安全性原則來降低這些風險，以建立初始治理策略。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-113">Use this information to discuss specific technical risks with members of your IT and security teams and develop a baseline set of security policies for mitigating these risks to establish your initial governance strategy.</span></span>

<span data-ttu-id="1ebc5-114">**部署規劃**：在部署任何工作負載或資產之前，請先執行安全性檢閱來找出任何新的風險，並確定已符合所有的存取和資料安全性原則需求。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-114">**Deployment planning**: Prior to deployment of any workload or asset, perform a security review to identify any new risks and ensure all access and data security policy requirements are met.</span></span>

<span data-ttu-id="1ebc5-115">**部署測試**：在部署任何工作負載或資產時，與公司內安全性小組合作的雲端治理小組會負責檢閱部署，以驗證安全性原則的合規性。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-115">**Deployment testing**: As part of the deployment process for any workload or asset, the Cloud Governance team, in cooperation with your corporate security teams, will be responsible for reviewing the deployment to validate security policy compliance.</span></span>

<span data-ttu-id="1ebc5-116">**年度規劃**：每年執行高階的安全性基準策略檢閱。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-116">**Annual planning**: On an annual basis, perform a high-level review of Security Baseline strategy.</span></span> <span data-ttu-id="1ebc5-117">探索公司未來計劃的優先順序以及更新後的雲端採用策略，以找出可能提高的風險和其他新浮現的安全性需求。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-117">Explore future corporate priorities and updated cloud adoption strategies to identify potential risk increase and other emerging security needs.</span></span> <span data-ttu-id="1ebc5-118">同時利用這個機會檢閱最新安全性基準最佳做法，並且將其整合至您的原則和檢閱流程。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-118">Also use this time to review the latest Security Baseline best practices and integrate these into your policies and review processes.</span></span>

<span data-ttu-id="1ebc5-119">**每季檢閱與規劃**：每季檢閱安全性稽核資料和事件報告，以識別安全性原則中任何必要的變更。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-119">**Quarterly review and planning**: On a quarterly basis perform a review of security audit data and incident reports to identify any changes required in security policy.</span></span> <span data-ttu-id="1ebc5-120">在此流程中，請審視目前的網路安全性情勢以主動預料新興的威脅，並適當更新原則。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-120">As part of this process, review the current cybersecurity landscape to proactively anticipate emerging threats, and update policy as appropriate.</span></span> <span data-ttu-id="1ebc5-121">完成檢閱之後，請讓設計指引能與更新後的原則契合。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-121">After the review is complete, align design guidance with updated policy.</span></span>

<span data-ttu-id="1ebc5-122">此規劃流程也是評估您的雲端治理小組目前成員資格，以了解新的或演進原則相關差距和安全性相關風險的好時機。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-122">This planning process is also a good time to evaluate the current membership of your Cloud Governance team for knowledge gaps related to new or evolving policy and risks related to security.</span></span> <span data-ttu-id="1ebc5-123">邀請相關 IT 人員來參與檢閱及規劃，無論是作為小組的暫時技術顧問或永久成員。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-123">Invite relevant IT staff to participate in reviews and planning as either temporary technical advisors or permanent members of your team.</span></span>

<span data-ttu-id="1ebc5-124">**教育與訓練**：每兩個月提供訓練課程，以確定 IT 人員和開發人員跟上最新的安全性原則需求。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-124">**Education and Training**: On a bi-monthly basis, offer training sessions to make sure IT staff and developers are up-to-date on the latest security policy requirements.</span></span> <span data-ttu-id="1ebc5-125">在此流程中檢閱及更新任何文件、指導方針或其他訓練資產，以確保這些項目與最新的公司原則聲明同步。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-125">As part of this process review and update any documentation, guidance, or other training assets to ensure they are in sync with the latest corporate policy statements.</span></span>

<span data-ttu-id="1ebc5-126">**每月稽核和報告檢閱**：每月對所有雲端部署執行稽核，以確保它們與安全性原則持續保持一致。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-126">**Monthly audit and reporting reviews**: On a monthly basis, perform an audit on all cloud deployments to assure their continued alignment with security policy.</span></span> <span data-ttu-id="1ebc5-127">檢閱 IT 人員的安全性相關活動，並識別任何未在現行監視和強制程序中處理的合規性問題。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-127">Review security related activities with IT staff and identify any compliance issues not already handled as part of the ongoing monitoring and enforcement process.</span></span> <span data-ttu-id="1ebc5-128">此檢閱的結果會向雲端策略小組和每個雲端採用小組報告，以告知整體的原則遵循狀況。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-128">The result of this review is a report for the Cloud Strategy team and each cloud adoption team to communicate overall adherence to policy.</span></span> <span data-ttu-id="1ebc5-129">報告也會儲存以便用於稽核和法律用途。</span><span class="sxs-lookup"><span data-stu-id="1ebc5-129">The report is also stored for auditing and legal purposes.</span></span>
